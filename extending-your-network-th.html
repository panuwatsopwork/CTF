Task 1 Introduction to Port Forwarding

Port Forwarding คือกลไกสำคัญที่ทำให้ Application หรือ Service ภายในเครือข่ายส่วนตัว สามารถ "เปิดประตู" ให้คนภายนอกอินเทอร์เน็ตเข้ามาเชื่อมต่อได้ ถ้าไม่มี Port Forwarding บริการอย่าง Web Server จะใช้งานได้แค่ภายในเครือข่ายเดียวกันเท่านั้น (intranet)

🏠 เปรียบเทียบให้เห็นภาพ

คิดเหมือนบ้านหลังหนึ่งที่มีคนอาศัยอยู่ภายใน ถ้าคนนอกจะมาส่งของ ต้องมี เลขที่บ้าน (Public IP) และ ประตูที่เปิดไว้ (Forwarded Port) เพื่อให้ของไปถึงห้องที่ถูกต้องในบ้าน

🌐 ตัวอย่างสถานการณ์

สมมุติว่ามี Web Server รัน Port 80 ที่ IP 192.168.1.10 เครื่องอื่นภายในเครือข่ายเดียวกันเข้าได้ แต่คนจากอินเทอร์เน็ตเข้าไม่ได้ ถ้า Admin ตั้งค่า Port Forwarding ที่ Router เครือข่าย Network #2 จะเข้าผ่าน Public IP ของ Network #1 (เช่น 82.62.51.70) ได้

🛡 Router

Router ทำ Port Forwarding ส่ง Request จากภายนอกไปเครื่องในเครือข่ายส่วนตัว

⚠️ Port Forwarding ≠ Firewall

Port Forwarding = "เปิดประตู" บอกว่าทราฟฟิกจะเดินไปไหน — Firewall = "ยามเฝ้าประตู" ตัดสินใจว่าทราฟฟิกจะผ่านได้หรือไม่ แม้ Port เปิดอยู่ Firewall ก็ยังบล็อกได้

🛠 อุปกรณ์ที่ใช้ตั้งค่า

Port Forwarding ตั้งค่าได้ที่ Router ของเครือข่าย โดยเข้าหน้าจัดการของ Router แล้วระบุว่า Port ไหนจะส่งต่อไปยัง IP ภายในไหน

Task 2 Firewalls 101

Firewall คือ "ด่านตรวจชายแดน" ของเครือข่าย ทำหน้าที่ตัดสินใจว่า ทราฟฟิกใดจะเข้า/ออกได้บ้าง ผู้ดูแลระบบสามารถสร้างกฎ (rule) เพื่ออนุญาตหรือปฏิเสธทราฟฟิกได้ตามเงื่อนไขต่าง ๆ

🔎 Firewall ตรวจสอบอะไรบ้าง?

  • Source — ทราฟฟิกมาจากเครือข่าย/IP ไหน?
  • Destination — ทราฟฟิกจะไปเครือข่าย/IP ไหน?
  • Port — มุ่งหน้าไปที่ Port ใด (เช่น 80, 22, 3389)?
  • Protocol — เป็น TCP, UDP หรือทั้งคู่?

กระบวนการตรวจนี้เรียกว่า Packet Inspection

📏 Firewall ทำงานที่ Layer ไหน?

Firewall ทำงานครอบคลุม OSI Layer 3 และ Layer 4 (Network + Transport) — จึงเข้าใจทั้ง IP Address และ Port/Protocol

🏷 ประเภทของ Firewall

ประเภทหลักการทำงานข้อดี / ข้อเสีย
Stateful ดู "บริบทของการเชื่อมต่อทั้งหมด" ไม่ใช่แค่ Packet ทีละชิ้น เช่น ติดตาม TCP Handshake ฉลาดกว่า ตรวจจับพฤติกรรมได้ แต่กิน Resource สูง
ถ้า host หนึ่งทำตัวไม่ดี → block ทั้งเครื่องเลย
Stateless ดูเฉพาะ Packet เดี่ยว ๆ ตามกฎตายตัวที่ตั้งไว้ เร็ว ใช้ Resource น้อย แต่ "โง่" หน่อย
เหมาะรับทราฟฟิกปริมาณมาก เช่น ป้องกัน DDoS

💡 Firewall มีหลายรูปแบบ

มีตั้งแต่ Hardware ขนาดใหญ่ในองค์กร ไปจนถึง Software เช่น Snort หรือแม้แต่ Router ที่บ้านก็มี Firewall ในตัว

ใน Firewall ลำดับกฎสำคัญมาก — ถ้ามี "allow all" อยู่ก่อน rule บล็อก ทุกอย่างจะถูกอนุญาตหมดและกฎบล็อกจะไม่ทำงาน

Task 4 VPN Basics

VPN (Virtual Private Network) คือเทคโนโลยีที่สร้าง "อุโมงค์เข้ารหัส (Tunnel)" บนอินเทอร์เน็ต ทำให้อุปกรณ์ที่อยู่คนละเครือข่ายสามารถสื่อสารกันได้ เสมือนอยู่ในเครือข่ายเดียวกัน

🏢 ตัวอย่าง: เชื่อมออฟฟิศ 2 ที่

บริษัทมีออฟฟิศ A (Network #1) และออฟฟิศ B (Network #2) คนละจังหวัด — สร้าง VPN ขึ้นมาเป็น Network #3 ทำให้อุปกรณ์ของทั้งสองที่คุยกันได้เหมือนนั่งทำงานอยู่ออฟฟิศเดียวกัน

✨ ประโยชน์ของ VPN

ประโยชน์รายละเอียด
เชื่อมเครือข่ายข้ามที่ตั้งหลายออฟฟิศคนละประเทศ ก็ใช้ทรัพยากร เช่น Server ร่วมกันได้
ความเป็นส่วนตัว (Privacy)เข้ารหัสข้อมูล ป้องกัน sniffing บน WiFi สาธารณะที่ไม่ปลอดภัย
ความนิรนาม (Anonymity)นักข่าว/นักเคลื่อนไหวใช้ปกป้องตัวเองในประเทศที่จำกัดเสรีภาพ — แต่ปลอดภัยแค่ไหนขึ้นกับนโยบาย Log ของผู้ให้บริการ

🔧 เทคโนโลยี VPN ที่ควรรู้จัก

PPP
(Point-to-Point Protocol)		 ทำหน้าที่ Authentication + Encryption ของข้อมูล ใช้ Private Key + Public Certificate (คล้าย SSH) — แต่ ออกจากเครือข่ายเองไม่ได้ (non-routable) ต้องอาศัย PPTP

(Internet Protocol Security)		 เข้ารหัสบน IP Framework โดยตรง ปลอดภัยสูง รองรับหลายอุปกรณ์ — แต่ติดตั้งยากกว่า

Task 5 LAN Networking Devices — Router & Switch

🛜 Router คืออะไร?

Router ทำหน้าที่ "เชื่อมเครือข่ายเข้าด้วยกัน" และส่งข้อมูลข้ามเครือข่าย (Routing) ทำงานที่ OSI Layer 3 มี UI (เว็บ/Console) สำหรับตั้งค่า เช่น Port Forwarding, Firewall

🗺 การเลือกเส้นทาง (Routing)

เมื่อมีหลายเส้นทางไปยังปลายทางเดียวกัน Router จะใช้ Protocol เลือกเส้นที่ดีที่สุดโดยพิจารณา:

  • เส้นทางที่สั้นที่สุด
  • เส้นทางที่เชื่อถือได้มากที่สุด
  • สื่อกลางที่เร็วที่สุด (เช่น Fiber เร็วกว่า Copper)

🔌 Switch คืออะไร?

Switch เป็นอุปกรณ์ที่ให้หลายเครื่อง (3–63) เชื่อมต่อกันด้วยสาย Ethernet — ทำงานได้ทั้ง Layer 2 และ Layer 3 (แต่ Switch Layer 2 จะไม่สามารถทำงาน Layer 3 ได้)

ประเภท Switchหน้าที่
Layer 2 Switchส่ง Frame ไปยังเครื่องปลายทางโดยอ้างอิง MAC Address
Layer 3 Switchทำได้ทั้งส่ง Frame ตาม MAC และ Route Packet ด้วย IP Protocol — เหมือน Switch ผสม Router

🧩 VLAN — Virtual LAN

VLAN คือเทคโนโลยีที่ "แบ่งเครือข่ายเสมือน" ใน Switch ตัวเดียวกัน เครื่องในแต่ละ VLAN จะถูกมองเป็นคนละเครือข่าย แม้จะต่อสายเข้ากับ Switch เครื่องเดียวกันก็ตาม

🏢 ตัวอย่างการใช้งาน VLAN

แผนก Sales และ Accounting ต่อ Switch เดียวกัน — ออก Internet ได้ทั้งสองแผนก แต่ คุยกันข้ามแผนกไม่ได้ เพราะอยู่คนละ VLAN เพิ่มความปลอดภัยและบริหารจัดการง่าย

🧠 Key Takeaways

  • Port Forwarding = "เปิดประตู" จากภายนอกเข้ามาในเครือข่าย ตั้งค่าที่ Router
  • Firewall = "ยามเฝ้าประตู" ตัดสินใจอนุญาต/บล็อก ตามกฎที่ตั้ง — มี Stateful (ฉลาด/ช้า) และ Stateless (เร็ว/โง่)
  • VPN = อุโมงค์เข้ารหัส (Tunnel) เชื่อมเครือข่ายข้ามอินเทอร์เน็ต — PPP, PPTP, IPSec
  • Router (L3) เชื่อมเครือข่ายและเลือกเส้นทาง — Switch (L2/L3) เชื่อมอุปกรณ์ในเครือข่ายเดียวกัน
  • VLAN แบ่ง LAN เสมือน เพิ่ม Security แม้ใช้ Switch ตัวเดียว

🚀 นำไปต่อยอด

💼 Cybersecurity

  • วิเคราะห์ Firewall Rule เพื่อหา Misconfiguration
  • ใช้ VPN เพื่อ Pentest อย่างปลอดภัย
  • ตรวจ VLAN hopping ในเครือข่ายเป้าหมาย

👨‍💻 Network Admin

  • ออกแบบ Network Topology ที่ทั้งปลอดภัยและรองรับ Scale
  • ตั้ง Site-to-Site VPN เชื่อมหลายสาขา
  • ใช้ VLAN แยกแผนกตาม Security Policy

📝 Checklist ก่อนไปบทเรียนถัดไป

  • อธิบายได้ว่า Port Forwarding ต่างจาก Firewall อย่างไร
  • บอกความต่างระหว่าง Stateful และ Stateless Firewall ได้
  • เข้าใจประโยชน์ของ VPN และเทคโนโลยี PPP / PPTP / IPSec
  • แยก Router (L3) กับ Switch (L2/L3) ได้
  • เข้าใจว่า VLAN ช่วยเรื่องความปลอดภัยอย่างไร
